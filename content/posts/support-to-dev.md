I spent four years supporting Zoho Invoice and Subscriptions before I built applications full-time. That detour turned out to be an education.

## You learn what actually breaks

Support shows you the edge cases developers never imagine — the weird inputs, the misread instructions, the workflows nobody designed for. You build differently once you've answered those tickets.

## Empathy is a skill

Explaining software to frustrated users teaches you to think from their side. That instinct carries straight into design decisions.

## Ship for the person, not the spec

The best thing support taught me: software succeeds when a real person's day gets easier — not when a requirement is technically met.
